# 6.7 板级联调验证

板级联调验证的目的在于确认硬件接口链路和后级显示链路能够实际运行。受实验条件限制，当前阶段尚未将换能器安装到管道上开展实流标定，因此本节不评价完整流量计在真实管道工况下的测量精度，而重点验证“FPGA 侧特征包输出 - MCU 接收解析 - 后级时间差与流量计算 - LCD 显示”这一数据链路的可实现性。

在 FPGA 侧，针对特征包输出逻辑开展 Vivado 仿真。仿真中构造固定的 `22 Byte` 特征数据包，并观察 SPI 输出端的片选、时钟、数据以及中断提示等信号。结果表明，在数据包准备完成后，FPGA 能够按照既定字节顺序从 SPI 输出端送出特征数据，说明接口输出逻辑已经具备向 MCU 提供固定格式特征包的能力。该验证覆盖的是 FPGA 输出接口和包组织逻辑，而不是装管条件下的完整回波采集与相关处理链路。

在板级联调中，FPGA 通过 SPI 向 MCU 发送固定特征数据包，MCU 按照 `idxA`、`idxB`、`y1`、`y2`、`y3` 的字段格式完成接收和解析，并调用后级算法计算传播时间差、流速和流量。计算结果最终在 LCD 屏幕上刷新显示。该过程验证了从 FPGA 接口输出到 MCU 算法处理再到人机显示的完整后级链路，说明前后级接口格式、SPI 读取流程、数据解析方式和 LCD 显示流程之间能够正确衔接。

为了使该部分验证更完整，可补充以下实验结果。

| 实验内容 | 建议展示结果 | 说明 |
| --- | --- | --- |
| Vivado SPI 输出仿真 | `cs/sclk/mosi/fpga_int` 等信号波形，以及 `22 Byte` 输出字节序列 | 证明 FPGA 侧固定特征包能够从 SPI 输出端按时序发出 |
| MCU 接收解析验证 | 发送字段值与 MCU 解析字段值对照表 | 证明 `idxA`、`idxB`、`y1`、`y2`、`y3` 的字节拼接和符号恢复正确 |
| 后级算法输出验证 | 固定特征包对应的 `dt`、流速、流量计算结果表 | 证明 MCU 接收到特征后能够进入后级解算链路 |
| LCD 显示验证 | LCD 显示照片或串口日志截图 | 证明计算结果能够在显示端实时刷新 |
| 多组固定包验证 | 低、中、高三组特征包及对应显示结果 | 证明输出结果随输入特征变化而变化，而非固定显示 |
| 连续发送稳定性验证 | 连续发送次数、解析错误次数、显示刷新是否异常 | 证明 SPI 接口和 MCU 解析流程具有基本稳定性 |

在当前验证条件下，板级联调结论应限定为：系统已经完成固定特征数据包从 FPGA 侧 SPI 输出到 MCU 接收解析、流量计算和 LCD 显示的一整条链路验证；结合前述 MATLAB 仿真和 Vivado 接口仿真，可以说明本文设计的前后级数据接口和后级解算显示流程具有可实现性。由于尚未完成管道安装与真实流体条件下的标定测试，本文不对装管实流精度、长期运行漂移和复杂工况适应性作最终评价，这部分可作为后续工程验证内容。
