# 第六章仿真与测试执行说明

本文档用于指导第六章实验的实际推进。目标不是一次性做出所有高阶仿真，而是先建立一条能够稳定产出论文图表和表格的最小可行验证链，再逐步补充噪声分析、对比实验和板级联调内容。

## 一、先做什么

第六章建议按下面顺序推进：

1. 先完成 MATLAB 特征级仿真平台。
2. 先做无噪声正确性验证。
3. 再做含噪声条件下的 `dt` 误差分析。
4. 再做流量输出误差分析。
5. 再做有无插值、有无后级滤波的对比。
6. 最后整理板级联调记录和截图。

这一顺序的好处在于，前四步就足以支撑第六章主体，后两步则作为增强内容继续补强说服力。

第一版仿真建议直接采用当前工程默认参数，不要一开始就改动太多。推荐先固定：

```matlab

同时建议固定一组测试流速点，例如：

```

这一组流速点已经足够覆盖低流速、中等流速和相对较高流速区间。

## 四、第一阶段必须完成的实验

### 1. 无噪声下的时差正确性验证

这一组实验的目的是回答：

“在理想条件下，`idxA/idxB/y1/y2/y3 -> dt` 这条链路是否正确？”

你需要做的事：

1. 选定一组 `v_true`。
2. 用当前代码中的逆推关系生成 `dt_true`。
3. 再由 `dt_true` 生成 `idxA/idxB/y1/y2/y3`。
4. 按 MCU 当前算法恢复 `dt_est`。
5. 比较 `dt_est` 与 `dt_true`。

建议输出：

- `dt_true` 与 `dt_est` 对比曲线
- `dt` 绝对误差表
- `dt` 相对误差表

建议论文落点：

“在无噪声条件下，特征包构造与后级 `dt` 解算链能够正确恢复传播时间差。”

### 2. 无噪声下的流量正确性验证

这一组实验回答：

“由 `dt` 进一步换算得到的流速和流量结果是否正确？”

你需要做的事：

1. 沿用上一组 `dt_est`。
2. 用当前工程公式计算 `v_est` 和 `Q_est`。
3. 与 `v_true`、`Q_true` 比较。

建议输出：

- `v_true` 与 `v_est` 对比图
- `Q_true` 与 `Q_est` 对比图
- 各流速点误差表

建议论文落点：

“在理想条件下，MCU 后级换算链能够将传播时间差稳定映射为流速和流量结果。”

## 五、第二阶段重点实验

### 3. 噪声条件下的 `dt` 鲁棒性分析

这一组实验最重要，因为它最直接体现“时差测量模块”的抗干扰能力。

建议做法：

1. 固定一个或多个代表性流速点，例如 `0.5 m/s`、`1.0 m/s`、`2.0 m/s`。
2. 设置一组噪声等级或 `SNR`，例如 `40, 30, 20, 10 dB`。
3. 每个 `SNR` 条件下重复 `100~300` 次随机实验。
4. 统计 `dt` 误差。

建议输出：

- `SNR-Bias(dt)` 曲线
- `SNR-STD(dt)` 曲线
- `SNR-RMSE(dt)` 曲线
- `SNR-离群率(dt)` 曲线

建议论文落点：

“噪声增强后，时差估计偏差和波动增大，但在一定 `SNR` 范围内仍保持可接受精度。”

### 4. 噪声条件下的流量输出误差分析

这一组实验回答：

“时差测量误差最终会怎样影响流量输出？”

做法与上一组类似，只是把最终评价量换成 `v_est` 或 `Q_est`。

建议输出：

- `SNR-RMSE(v)` 曲线
- `SNR-RMSE(Q)` 曲线
- `SNR-95%置信区间` 图

建议论文落点：

“虽然误差起源于 `dt`，但经后级换算后仍可观察到相应流量偏差，因此第六章需要同时给出 `dt` 误差和流量误差。”

## 六、第三阶段增强实验

### 5. 有无插值的对比

这一组实验非常值得做，因为它能直接支撑你第四章和第五章关于三点特征必要性的论证。

建议分两组：

1. 只用整数索引，不用 `y1/y2/y3`。
2. 使用整数索引加三点抛物线插值。

比较指标：

- `dt` 的 `RMSE`
- `v` 的 `RMSE`
- 低流速点误差放大图

建议论文结论方向：

“在低流速和小时间差条件下，引入三点插值后，时间差与流量估计精度均明显改善。”

### 6. 有无后级滤波的对比

这一组实验用于支撑 `5.5` 的后处理设计。

建议分三组：

1. 不做平滑
2. 仅做 IQR 滑动窗口
3. IQR + 卡尔曼滤波

比较指标：

- 输出波动幅度
- 标准差
- 响应速度

建议论文结论方向：

“后级稳定化处理并不改变测量主链原理，但能够显著改善输出结果的平稳性。”

## 八、统计指标怎么定义

第六章建议至少统一使用下面四个指标：

1. `Bias`
   误差均值，反映是否存在系统性偏大或偏小。
2. `STD`
   误差标准差，反映结果波动程度。
3. `RMSE`
   综合考虑偏差和波动，是最常用的整体误差指标。
4. `离群率`
   误差超过阈值的比例，反映鲁棒性。

如果时间允许，可以再补：

5. `95% 置信区间`

## 九、板级联调要做什么

MATLAB 仿真之外，第六章还要留一节给板级联调。这里不追求像标准流量装置那样给出完整实流标定，而是证明链路已经打通。

建议至少记录：

1. FPGA 是否稳定输出特征包。
2. MCU 是否稳定收包并完成解算。
3. 结果是否能够持续刷新。
4. 多次运行是否稳定。

建议保留的材料：

- 串口日志截图
- 调试界面截图
- 一组或几组典型输出数据
- 若有逻辑分析仪或示波器图，也可保留
