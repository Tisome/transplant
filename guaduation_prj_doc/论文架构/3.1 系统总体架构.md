# 3.1 系统总体架构

本系统以超声波时差测量为核心，面向液体流量测量需求，构建了一套由前端信号采集、前级特征提取、后级时间差解算以及结果输出组成的完整测量链路。系统设计目标是在保证传播时间差提取精度的基础上，实现流量结果的稳定输出，并兼顾实际工程应用中的实时性、可扩展性和实现复杂度。

从整体组成来看，系统主要包括超声换能器、激励与接收电路、模数转换模块、FPGA 处理模块、MCU 控制与解算模块以及显示与通信模块等部分。其中，FPGA 不仅承担前级特征提取任务，还负责产生用于驱动换能器的激励控制信号，并与后续采样过程保持时序同步；激励与接收电路完成信号驱动、放大与整形，模数转换模块负责将模拟回波信号转换为数字信号，MCU 模块负责后级时间差解算、流量换算和参数管理，而显示与通信模块则用于结果展示和工程调试。

系统的基本工作流程可以概括为：首先由 FPGA 内部 PWM/激励控制逻辑产生驱动信号，通过前端电路作用于超声换能器，使其按设定时序发射超声波；超声波沿既定声路在流体中传播并被对应换能器接收后，FPGA 再发出同步采样触发信号，控制 ADC 对回波信号进行采样；随后，采样数据进入 FPGA 内部缓存与前级处理链路，完成相关运算、峰值搜索和关键特征提取；再由 MCU 对 FPGA 输出的特征数据进行解析，完成传播时间差解算，并进一步换算得到流速和流量结果；最后，测量结果通过显示界面或通信接口输出。根据现有设计，系统以前后两个 4 ms 子过程构成一个完整测量循环，即约 8 ms 完成一组顺逆流相关数据的获取与处理。由此，系统形成了从激励、采样、特征提取到结果输出的完整处理闭环。

在总体架构设计上，系统采用 FPGA 与 MCU 协同的实现方式。其中，FPGA 主要负责前端高速、并行、确定性强的处理任务，包括激励时序控制、采样触发、数据缓存、相关运算、峰值搜索和特征输出；MCU 则主要负责后级时间差解算、流量计算、参数配置、结果管理以及工程辅助功能。这样的任务划分能够充分发挥 FPGA 在高速数字信号处理和精确时序控制方面的优势，同时保留 MCU 在控制逻辑、算法调整和系统扩展方面的灵活性，从而在测量精度、系统实时性和工程可实现性之间取得平衡。

总体而言，本系统采用“前级特征提取 + 后级时间差解算”的分层设计思路，将复杂的原始采样信号处理任务前移至 FPGA，将与参数管理和结果输出相关的任务保留在 MCU 中，不仅有利于提升系统整体处理效率，也为后续模块化设计和功能扩展提供了基础。后续章节将在此基础上，进一步分析 FPGA 与 MCU 的任务划分、系统数据流程以及关键特征接口设计。
