# 3.2 FPGA 与 MCU 的任务划分

超声波时差测量同时要求较高的时间分辨率、稳定的实时响应以及较好的工程可实现性。若将前端时序控制、原始数据处理和后端解算管理全部集中于单一处理平台，往往难以同时兼顾处理速度、时序确定性和实现复杂度。基于这一考虑，本文采用 FPGA 与 MCU 协同的分层实现方式。

FPGA 主要承担与原始信号直接相关、且对时序约束较强的任务，包括激励控制、采样同步、数据缓存、卷积或相关计算、峰值搜索以及关键特征提取等。上述任务具有数据量大、并行性强、处理节拍固定等特点，适合在 FPGA 上实现。通过将高吞吐、强时序约束的处理前移至 FPGA，系统能够在较短时间内完成单帧数据预处理，并减少原始波形跨平台传输带来的负担。

MCU 主要承担后端解算与系统管理任务。FPGA 输出的关键特征进入 MCU 后，由其完成数据解析、插值细化、时间差求取、流量换算以及结果管理。相较于 FPGA，MCU 更适合实现参数配置、状态控制、结果平滑和外围接口扩展等功能，因此可作为后级控制与输出平台。

这种分工的关键在于：将“必须高速完成且需严格按时序执行”的部分放在 FPGA 上实现，将“需要灵活组织与便于调整”的部分放在 MCU 上实现。系统并不上传完整采样波形，而是由 FPGA 先提取关键特征，再由 MCU 完成后级解算，从而降低跨平台数据压力，并使前后级边界更加清晰。这一分层思路为后续系统优化与功能扩展提供了明确基础。
