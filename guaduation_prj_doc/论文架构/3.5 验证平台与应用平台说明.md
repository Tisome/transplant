# 3.5 验证平台与应用平台说明

为保证系统设计、实现与验证之间具有明确对应关系，本文对开发联调平台、软件工具环境以及最终演示样机作区分说明。由于本课题处于毕业设计研究阶段，系统验证与答辩展示并非完全在同一硬件平台上完成，因此有必要明确各平台的用途与定位。

在开发与联调阶段，MCU 侧功能验证主要在 `GD32F303RCT6` 开发板上完成，FPGA 侧硬件测试主要依托 `ATK-DF7020P` FPGA 开发平台开展。前者用于后级解算、接口联调与功能验证，后者用于 FPGA 逻辑下载、时序验证及前端处理链路测试。通过上述两类平台的配合，可以完成系统核心处理链的联调验证。

在软件开发与调试环境方面，FPGA 侧代码编写、综合与仿真主要在 `Vivado 2019.1` 环境下完成；MCU 侧软件开发采用 `Visual Studio Code`、`EIDE` 与 `Keil ARMCC` 组成的工具链，并结合 `SEGGER Ozone` 对关键变量和运行过程进行在线调试。上述工具环境分别服务于逻辑设计验证与嵌入式软件调试，为模块级和接口级验证提供支持。

在答辩展示阶段，系统采用一块已完成集成的样机板进行演示。该样机以 `GD32F303CCT6` 和 `XC7S25CSGA225-1` 为核心器件，用于展示方案在目标硬件形态下的整体运行效果。需要说明的是，论文中的主要验证结论仍以前期开发与联调平台所得结果为依据，而样机主要用于体现方案的工程集成与部署能力。

总体而言，平台划分的变化并不改变系统的核心测量原理与前后级分层结构。开发板阶段承担方案验证任务，样机阶段承担集成展示任务，二者共同构成了本文的实现与验证基础。
