## 题目对应的摘要草稿

超声波时差法具有非接触、无压损和响应速度快等特点，在流体测量领域具有重要应用价值。针对超声波顺流与逆流传播时间差微小、传统处理平台难以兼顾高时间分辨率与实时性的问题，本文围绕《基于FPGA的超声波时差测量模块》这一题目，设计并实现了一种基于 FPGA 的超声波时差测量方案。

本文采用 FPGA 与 MCU 协同架构，其中 FPGA 负责前级高速数据处理与时差相关特征提取，MCU 负责特征包接收、传播时间差解算及后续流量输出。针对超声信号处理过程中峰值定位精度不足的问题，系统采用峰值索引与三点抛物线插值相结合的方式进行时间差估计，并在后级加入去异常、滤波和平滑处理，以提高结果稳定性。当前验证阶段以 `GD32F303RCT6` 为主控平台完成链路联调，后续应用面向 `GD32F303CCT6` 平台进行移植。

考虑到当前阶段尚未完成完整装管条件下的标准流量装置实测，本文采用 MATLAB 仿真与板级联调相结合的方式进行验证。仿真部分重点验证时差测量模块的正确性与抗噪性能，并在此基础上进一步分析流量输出结果；联调部分验证 FPGA、MCU 与软件处理链的工作可行性。结果表明，本文所设计的超声波时差测量模块能够实现从特征提取到时间差解算的完整处理流程，并具备进一步用于流量测量系统的实现基础。

关键词：FPGA；超声波时差测量；抛物线插值；流量输出；MATLAB 仿真

## 第一章 绪论

### 1.1 研究背景与研究意义

超声波测量技术因其非接触、适应性强和实时性较好等特点，在流体检测领域得到广泛关注。其中，时差法通过比较超声波在顺流与逆流方向上的传播时间差来反映流体运动状态，是超声波测流中较为典型的一类方法。对于该方法而言，传播时间差通常较小，因此系统能否稳定、准确地提取时间差，是决定后续测量结果质量的关键。

在传统实现方式中，若仅依赖 MCU 对高频采样数据进行处理，往往会在时间分辨率、数据吞吐量和实时性方面受到限制。相比之下，FPGA 具有并行处理能力强、时序控制精确和逻辑结构灵活可配置等特点，更适合承担前级高速特征提取任务。因此，研究基于 FPGA 的超声波时差测量模块，不仅具有理论意义，也具有较强的工程应用价值。

需要指出的是，本文的研究重点并不是完整产品层面的通信、显示或菜单功能，而是围绕“如何稳定提取超声波传播时间差”展开。传播时间差作为中间量，在后续可进一步换算为流速和流量，因此时差测量模块既是本文的研究核心，也是整个后续测量链路的基础。

### 1.2 国内外研究现状

围绕超声波时差测量，国内外研究主要集中在以下几个方面：一是传播模型与测量原理研究，重点分析顺流与逆流传播时间差同流动状态之间的关系；二是时间差提取算法研究，包括阈值法、相关分析法及插值优化方法；三是系统实现研究，重点关注高分辨率时间测量、前端噪声抑制以及嵌入式平台协同处理等问题。

从现有研究可以看出，超声波时差测量的难点主要不在于后级流量公式本身，而在于前级时间差是否能够被稳定提取。对于噪声较强或峰值不明显的信号，仅依赖简单阈值检测容易导致测量误差增大，因此相关峰值提取和亚采样插值成为重要研究方向。随着 FPGA 等平台的发展，将前级高频处理任务前移到硬件逻辑中完成，已成为较为合理的实现思路。

总体而言，超声波时差测量研究已经具有较好的理论基础，但在工程实现层面仍然需要进一步解决前级特征提取、后级稳定化处理以及系统验证方法等问题。本文即是在这一背景下，面向时差测量模块本身展开设计与实现。

### 1.3 本文主要工作

本文围绕基于 FPGA 的超声波时差测量模块开展研究，主要完成以下几个方面的工作。

第一，完成了系统总体架构设计，明确 FPGA 与 MCU 的分工关系。FPGA 负责前级高速时差信号特征提取，MCU 负责特征包解析、时间差解算以及后续流量输出。

第二，设计并实现了基于峰值索引和三点抛物线插值的时间差估计方法，使系统能够在离散采样点基础上获得更细致的亚采样时间差估计结果。

第三，基于现有工程实现，构建了从特征包到时间差、再到流量输出的完整软件处理链，并加入了信号质量评估、去异常和平滑处理过程，以提高结果稳定性。

第四，结合当前实验条件，提出了以 MATLAB 仿真和板级联调为主的验证方案，用于验证时差测量模块设计的正确性和可行性。当前验证阶段采用 `GD32F303RCT6` 作为联调平台，后续面向 `GD32F303CCT6` 开展应用移植。

## 第三章 系统总体方案设计

### 3.1 系统总体设计思路

本文设计的系统以超声波时差测量模块为核心，整体采用“前级特征提取 + 后级物理量解释”的协同方案。系统并不直接在后级处理全部原始采样数据，而是由 FPGA 先对高速信号进行预处理和特征压缩，再由 MCU 根据特征结果完成时间差解算和流量输出。这样既突出了时差测量模块的研究重点，也避免了原始高频数据直接压向 MCU 所带来的计算压力。

从系统链路来看，超声信号经前端采样后进入 FPGA。FPGA 不直接输出最终流量，而是输出与时差估计相关的关键特征信息，包括峰值索引和峰值附近三点相关值。MCU 接收这些特征后，根据采样周期和插值方法计算传播时间差，并进一步换算为流速和流量。因此，整个系统的核心思想可以概括为：前级重点解决“怎么测到时间差”，后级重点解决“怎么把时间差解释成可输出结果”。

### 3.2 FPGA 与 MCU 的任务划分

在本文方案中，FPGA 的主要职责包括采样数据缓存、卷积或互相关特征提取、峰值索引提取以及特征包输出。其核心目标是在高速采样条件下，稳定地完成与传播时间差相关的特征计算。

MCU 的主要职责包括 SPI 接收、特征包解析、时间差解算、流量换算、结果稳定化处理以及工程化输出。这里需要强调的是，MCU 端虽然最终会输出流量，但它所依赖的基础仍然是 FPGA 端提供的时差特征结果。因此，从论文写作角度看，MCU 的角色更适合作为“时差模块的后级解释与应用扩展”，而不是全文研究重点的替代物。

### 3.3 工程化功能的写法

为便于调试和工程演示，系统进一步实现了基于 Modbus 的数据通信和基于 LVGL 的结果显示功能。但这部分内容在本文中只作为工程辅助功能简要介绍，不作为主要研究重点展开。更准确的写法是：Modbus 和 LVGL 的引入主要服务于测量结果观察、调试和系统联调，而不是论文的核心创新点。

## 第四章 FPGA 超声波时差测量模块设计

### 4.1 模块设计目标

FPGA 时差测量模块的设计目标是：在高速采样条件下，从超声信号中提取能够稳定反映传播时间差的关键特征量，并将这些特征量以较低的数据开销传递给后级 MCU。与直接传输整段采样波形相比，该方案能够显著降低通信压力，并使后级软件处理更聚焦于时间差解算和物理量输出。

### 4.2 特征量设计思路

本文方案中，FPGA 输出的关键特征包括峰值索引 `idxA/idxB` 以及峰值附近三点相关值 `y1/y2/y3`。其中，峰值索引用于提供传播时间差的粗定位信息，而三点相关值则用于后级抛物线插值，从而补偿离散采样点带来的分辨率限制。该设计兼顾了时差估计精度和硬件实现复杂度，能够在保持较低接口开销的同时，为后级高精度解算提供足够的信息支持。

### 4.3 为什么要这样做

若仅使用峰值索引进行时间差估计，则结果会受到采样周期限制，难以获得更细致的亚采样分辨率；若在 FPGA 内部直接完成全部高精度插值运算，又会增加实现复杂度。因此，本文采用“FPGA 负责粗定位和局部特征提取，MCU 负责后级插值解算”的分工方式。这种设计既发挥了 FPGA 在高速并行处理方面的优势，又保留了 MCU 在后级算法调整上的灵活性，属于面向时差测量任务的一种工程上较为平衡的实现方案。

## 第五章 MCU 端时间差解算与流量输出

### 5.1 时间差解算流程

MCU 端首先对 FPGA 输出特征包进行解析，获得峰值索引和三点相关值。随后，根据采样周期将索引值换算成时间基准，再利用三点抛物线插值方法求得峰值在离散采样点之间的偏移量。最终，系统将整数索引差与亚采样偏移量相结合，得到更精细的传播时间差估计结果。

这种处理方式的优势在于：一方面，时间差的粗定位信息已经由 FPGA 提前提取出来；另一方面，后级插值计算不需要直接处理整段高速波形，从而降低了 MCU 端实现难度。因此，MCU 端的核心工作并不是“重新做一次测量”，而是对 FPGA 提供的时差特征进行解释和精细化解算。

### 5.2 由时间差到流量输出

在获得传播时间差后，系统根据设定的几何参数和固定延时参数完成流速计算，并进一步根据管道横截面积换算得到流量。需要强调的是，在本文中流量输出属于后级应用结果，它的重要性在于能够反映时差模块对实际测量任务的支撑能力，但其前提始终是时间差测量过程本身足够可靠。

为提高输出稳定性，后级软件还包含去异常、滑动窗口平均、卡尔曼滤波和零漂补偿等处理过程。这些算法的目的并不是改变时差测量模块的本质，而是尽量减少异常点和波动对后续流量输出的影响。

本文的验证重点首先是时差测量模块是否能够正确提取传播时间差，其次才是基于该时间差所得到的流量输出是否合理。因此，在验证章节中应将“时间差测量正确性”作为第一层验证目标，将“流量输出正确性”作为第二层验证目标，并将板级联调作为工程可行性验证。
