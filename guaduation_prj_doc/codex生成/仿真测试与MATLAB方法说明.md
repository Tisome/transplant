# 仿真测试与 MATLAB 方法说明

## 一、先把重心摆正

你这篇论文题目是《基于FPGA的超声波时差测量模块》，所以仿真和测试的核心目标应该是：

1. 先证明“时间差测量模块”是对的
2. 再证明“基于该时间差得到的流量输出”是合理的
3. 最后证明“这套链路在板子上能跑起来”

也就是说，验证顺序应该是：

`dt 是否算对 -> Q 是否算合理 -> 系统是否能联调`

不是一上来就把重点放在显示、通信或者完整产品功能上。

## 三、最关键的几个概念

## 1. 互相关到底是什么

互相关可以理解成：

“把两个信号左右滑动，看看什么时候最像。”

如果两个信号只是有时间延迟，那么：

1. 没对齐时，相似度不高
2. 刚对齐时，相似度最高
3. 相似度最高的位置，就是我们要找的时延

所以互相关的用途就是：

“帮助找到时间差的大概位置。”

## 2. 为什么还需要抛物线插值

采样信号是离散的，峰值不一定刚好落在某一个采样点上。

如果只看整数采样点，那么时间差分辨率就被采样周期限制住了。

所以你工程里又做了一步：

1. 先用 `idxA/idxB` 找到峰值大概在哪
2. 再用 `y1/y2/y3` 做三点抛物线插值
3. 算出峰值相对整数采样点的偏移量 `delta`

最终再得到更细的时间差。

这一步在论文里非常值得写，因为它恰好回答了：

“为什么 FPGA 不直接把完整波形传给 MCU，而只传关键特征？”

## 3. SNR 是什么

SNR 就是信号和噪声的强弱对比。

你可以简单理解成：

1. SNR 高：信号清楚，时差更容易算准
2. SNR 低：噪声更重，峰值更容易抖动

所以论文里最常见的一类图，就是误差随 SNR 变化的曲线。

## 4. 蒙特卡洛是什么

它本质上就是“同样条件下多做几次随机实验，再看统计结果”。

例如：

1. 固定一个流量真值
2. 固定一个 SNR
3. 每次随机生成不同噪声
4. 重复 200 次
5. 统计平均误差和误差波动

如果你现在还没做过，也完全可以先不做，只做单次和少量重复验证也能先支撑论文骨架。

## 四、最推荐你的 MATLAB 仿真路线

对你当前项目，我最推荐先做“特征级仿真”，再考虑“波形级仿真”。

## 路线 A：特征级仿真

这是最适合你现在起步的。

### 它在做什么

不先模拟完整超声波形，而是直接从“理论时间差”构造 FPGA 会输出的特征。

具体流程是：

1. 给定理论流量 `Q_true`
2. 由流量算流速 `v_true`
3. 由流速反推理论时间差 `dt_true`
4. 由 `dt_true` 构造 `idxA/idxB/y1/y2/y3`
5. 再按你代码里的逻辑恢复出 `dt_est`
6. 再由 `dt_est` 算 `Q_est`

### 它的优点

1. 上手快
2. 最贴近你现在 MCU 端代码
3. 很适合先写论文验证章节

### 它能证明什么

它能证明：

1. 特征包设计是合理的
2. 后级时间差解算公式是正确的
3. 后级流量输出链是通的

## 路线 B：波形级仿真

等你路线 A 跑通后，再做这个。

### 它在做什么

1. 生成参考发射信号
2. 生成带时间延迟的接收回波
3. 给回波加噪声
4. 做相关处理
5. 提取峰值索引和三点值
6. 再走后级解算

### 它的意义

这会更像“完整前级测量过程”，但实现复杂度也更高。

如果你现在还不熟，完全可以先不做满。

## 五、最小可行 MATLAB 仿真步骤

下面给你一个最适合当前上手的版本。

## 第一步：确定基础参数

你现在代码里最关键的基础参数有：

```matlab
Ts_ns = 1e9 / 65e6;          % 采样周期
inner_diameter_mm = 20.0;    % 示例
wall_thick_mm = 1.0;
cos_value = 0.913545;
sin_value = 0.406738;
te_ns = 12000.0;
```

如果要算横截面积：

```matlab
D_m = inner_diameter_mm * 1e-3;
A = pi * (D_m/2)^2;
```

## 第二步：先给一个流量真值

建议先统一单位，用 `m^3/s`。

```matlab
Q_true = 1.0 / 3600.0;   % 例如 1 m^3/h
v_true = Q_true / A;
```

这样你就得到了理论流速。

## 第三步：由流速反推理论时间差

你现有代码的流速计算逻辑可以反着用。

核心关系可以理解为：

```text
v <- dt
```

所以仿真里可以改成：

```text
dt_true <- v_true
```

如果用你当前代码的思路，可以写成：

```matlab
t1_ns = 3000 * Ts_ns;
t_wall_ns = 0;           % 初始简化时可先设为 0
a = t1_ns - te_ns - t_wall_ns;
b = a;
cos_sin = cos_value * sin_value;
L1_mm = inner_diameter_mm;

v_mm_per_ns = v_true * 1e-6;
dt_true_ns = v_mm_per_ns * (cos_sin * a * b) / L1_mm;
```

如果你担心太复杂，第一版完全可以先把 `t_wall_ns` 简化掉。

## 第四步：把 `dt_true` 变成 `idx` 和 `delta`

```matlab
idxA = 3000;
d = dt_true_ns / Ts_ns;
di = floor(d + 0.5);
delta_true = d - di;
idxB = idxA + di;
```

含义是：

1. 整数部分给索引差
2. 小数部分给后面的插值偏移

## 第五步：构造 `y1/y2/y3`

这一步你完全可以直接沿用你项目里的构造思路：

```matlab
A0 = 1e9;
K = 1e8;

y2 = A0 - K * (0 - delta_true)^2;
y1 = A0 - K * (1 - delta_true)^2;
y3 = A0 - K * (-1 - delta_true)^2;
```

这本质上是在人工构造一个局部峰值。

## 第六步：如果要加噪声，就在这里加

最简单的是高斯噪声：

```matlab
noise_sigma = 5e5;
y1n = y1 + noise_sigma * randn;
y2n = y2 + noise_sigma * randn;
y3n = y3 + noise_sigma * randn;
```

第一版只做这一种就足够了。

## 第七步：按代码逻辑恢复 `dt`

```matlab
den = (y3n - 2*y2n + y1n);
delta_est = 0.5 * (y3n - y1n) / den;
dt_est_ns = ((idxB - idxA) + delta_est) * Ts_ns;
```

这一步就是你 MCU 端 `dt` 解算逻辑的 MATLAB 版本。

## 第八步：由 `dt_est` 算回流量

```matlab
v_est_mm_per_ns = dt_est_ns * L1_mm / (cos_sin * a * b);
v_est = v_est_mm_per_ns * 1e6;
Q_est = v_est * A;
```

这样你就得到了：

1. `dt_est`
2. `v_est`
3. `Q_est`

## 六、论文里最值得做的几组实验

## 实验 1：无噪声下的正确性验证

目的：

证明你的时间差测量链和流量输出链在理想条件下是对的。

做法：

1. 不加噪声
2. 给多组 `Q_true`
3. 比较 `Q_est`
4. 看误差是否很小

这一组实验最适合用来写“基础正确性验证”。

## 实验 2：不同 SNR 下的时间差误差分析

目的：

看噪声增强时，时差模块会不会变差。

做法：

1. 固定一个流量点
2. 设置多个噪声强度
3. 多次重复仿真
4. 统计 `dt` 误差

这一组实验直接对应论文题目，非常重要。

## 实验 3：不同 SNR 下的流量误差分析

目的：

说明时差测量误差最终会怎样影响流量输出。

做法：

1. 用上一组仿真得到的 `dt_est`
2. 再算 `Q_est`
3. 统计流量误差

这一组实验适合用来说明：

“虽然论文主角是时差模块，但它已经具备支撑后级流量输出的能力。”

## 实验 4：有无插值的对比

目的：

证明 `y1/y2/y3` 和抛物线插值不是多余设计。

做法：

1. 一组只用 `idxA/idxB`
2. 一组用 `idxA/idxB + y1/y2/y3`
3. 比较时间差误差和流量误差

这个实验很值得做，因为它最能体现你这个模块设计的必要性。

## 实验 5：有无后级滤波的对比

目的：

证明后级滤波和平滑是为了提高输出稳定性，而不是论文主线本身。

做法：

1. 不做滤波
2. 做滑动窗口
3. 做滑动窗口 + 卡尔曼
4. 做完整后处理链

注意论文里要写清楚：

“这部分属于时差模块后级应用补充，用于改善流量输出稳定性。”

## 九、板级测试到底测什么

因为你的论文主线不是 Modbus 和 LVGL，所以板级测试不要写偏。

板级测试真正要测的是：

1. FPGA 特征包有没有稳定输出
2. MCU 能不能稳定收包
3. `dt` 解算链能不能在板子上跑通
4. 流量结果有没有正常输出

`Modbus` 和 `LVGL` 可以作为辅助观察手段：

1. Modbus 用于把结果读出来
2. LVGL 用于把结果显示出来

但它们不是你测试的主体。
