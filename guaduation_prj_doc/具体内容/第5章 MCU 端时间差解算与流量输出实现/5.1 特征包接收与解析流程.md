# 5.1 特征包接收与解析流程

后级处理不再直接面对大规模原始采样波形，而是以定长特征包作为输入开展传播时间差与流量解算。因此，数据接收与解析的稳定性构成后续算法链路成立的前提条件。该部分工作的核心在于明确后级算法输入的来源、组成方式及其进入解算流程的组织形式。

按照接口定义，FPGA 向 MCU 提供的特征包总长度为 `22 Byte`，依次包含 `idxA`、`idxB`、`y1`、`y2` 和 `y3` 五部分。其中，`idxA` 与 `idxB` 对应两帧相关峰的粗定位索引，`y1`、`y2`、`y3` 对应峰值邻域三点相关值。由于数据结构固定且每次均以整包方式读出，MCU 侧可采用固定偏移解析策略，将数据接收过程与后续算法处理过程明确分离。

在所采用的接口组织方式中，数据读取由 MCU 以 SPI 主机模式主动完成，FPGA 则通过中断信号指示完整特征包已经准备完毕。当后级处于可接收状态时，就绪信号保持有效；一旦前端形成完整特征包，FPGA 即置位中断信号提示 MCU 发起读取。MCU 响应中断后完成 `22 Byte` 数据读取，随后恢复就绪状态。该组织方式保证后级始终以完整数据包为处理单位，从而避免因读取时刻不一致导致的数据碎片化问题。

数据接收完成后，MCU 需进一步完成字段类型恢复。`idxA` 与 `idxB` 通常按无符号索引量解释，用于后续整数采样点时间换算；`y1`、`y2`、`y3` 则需恢复为带符号局部相关值，以便参与抛物线插值运算。经由上述处理，后级获得了统一、低冗余且可直接用于解算的输入形式，后续传播时间差估计可由“粗定位索引 + 邻域三点值”这一特征组合展开。
