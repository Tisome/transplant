# 4.5 特征打包与接口输出设计

当 `idxA`、`idxB`、`y1`、`y2`、`y3` 完成提取后，前端处理链的最终环节即为将上述特征组织为稳定、统一的接口输出格式。该部分设计的重点不在于峰值获取过程本身，而在于如何保证关键特征以完整、一致且可直接利用的形式传递至后级平台。

按照所采用的接口定义，FPGA 输出的数据包长度固定为 `22 Byte`，其中 `idxA` 与 `idxB` 各占 `2 Byte`，`y1`、`y2`、`y3` 各占 `6 Byte`。由于 MCU 每次读取的数据格式完全固定，包边界可由片选信号与字节计数共同确定，因此接口设计未额外引入包头和校验字段，而采用定长包方式直接传输特征数据。相较于面向复杂网络环境的通用协议，该方案更适用于板级短距离 SPI 传输场景，能够在保持结构清晰的同时降低附加开销。

若以前端两帧完整采样波形作为传输对象，则单次测量需上传 `4000` 个 `16 bit` 采样值，对应数据量约为 `64000 bit`；而当前接口仅传输 `22 Byte`，即 `176 bit` 的特征包，压缩比约为 `64000/176≈364`。需要指出的是，这一压缩并非对原始数据的简单裁剪，而是保留了传播时间差解算所需的最小充分特征集合，即用于粗定位的 `idxA`、`idxB` 以及用于局部细化的 `y1`、`y2`、`y3`。因此，在显著降低总线负担的同时，后级仍可完成高分辨率时间差估计。

为保证单组特征在语义上的完整性，FPGA 并不会在单帧索引刚产生时立即输出数据，而是等待 `idxA`、`idxB`、`y1`、`y2`、`y3` 全部准备完成后，再统一写入待发缓存。由此，MCU 每次读取到的均为完整的 A/B 测量结果，而非来自不同周期的零散中间量。该组织方式降低了后级状态拼接复杂度，同时减小了跨周期读写导致数据不一致的风险。

在接口时序方面，系统采用由 FPGA 暂存结果、MCU 主动读取的从机输出方式。当 FPGA 内部形成完整特征包后，数据首先进入待发送状态；仅当 MCU 侧就绪信号保持有效时，FPGA 才置位中断提示并允许主机发起读取。随后，MCU 以 SPI 主机身份完成 `22 Byte` 数据读取，片选信号释放后本次传输结束，FPGA 清除中断并等待下一包数据。该机制有效解耦了前端测量节拍与总线读取节拍，从而避免后级尚未准备完毕时前端结果被提前暴露的问题。

总体而言，特征打包与接口输出并非单纯的数据格式约定，而是前后级协同边界的具体实现形式。FPGA 侧仅负责提供稳定、低冗余且可直接利用的关键特征，MCU 侧则基于固定格式完成插值、传播时间差解算及流量计算。正是由于接口边界被压缩为 `22 Byte` 定长特征包，整个系统才能在保证实时性的同时保持较好的模块独立性与扩展能力。
