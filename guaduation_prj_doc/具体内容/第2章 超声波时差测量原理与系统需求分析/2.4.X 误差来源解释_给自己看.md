# 2.4 误差来源解释（给自己看）

这份文档不是论文正文，而是把“2.4 时间差测量中的主要误差来源”里面提到的一些概念，换成更容易理解的话来说明，方便后面自己继续写论文和做验证。

## 1. 先把整个测量链路想清楚

你这个项目里，核心链路其实可以简单理解成下面这样：

`超声信号 -> FPGA 做相关处理 -> 找到峰值位置 -> 提取特征 -> MCU 算出 t1/t2/dt -> 换算流速和流量`

更具体一点，对应你现在的代码，大概是：

`idxA / idxB / y1 / y2 / y3 -> t1 / t2 / dt -> flow_speed -> flow_rate`

这里最关键的是 `dt`，因为：

1. `dt` 算不准，后面的流速就不准。
2. 流速不准，后面的流量也就跟着偏。

所以“误差来源分析”本质上就是在问：

“到底是哪些地方会把 `dt` 算歪？”

## 2. 什么叫“时间差提取过程”

“时间差提取过程”不是一个神秘的大词，本质上就是：

**从原始信号或特征量里，把顺流和逆流之间的时间差找出来的整个过程。**

在你的项目里，它并不是直接从原始波形里硬看，而是分成了两步：

1. FPGA 先做前级处理，给出峰值相关特征
2. MCU 再根据这些特征算出 `t1`、`t2` 和 `dt`

所以这里的“时间差提取过程”包括：

1. 相关运算
2. 峰值搜索
3. 特征输出
4. 峰值位置细化
5. `dt` 计算

只要这几步里任何一步有偏差，最后的 `dt` 就会有误差。

## 3. 什么叫“峰值搜索”

你可以把互相关结果想成一条曲线，这条曲线会在“两个信号最对齐”的地方出现一个最高点。

这个最高点的位置，就是你最关心的时间差位置。

“峰值搜索”就是：

**在这一条相关曲线里，找到最高点或极值点的位置。**

你现在代码里的 `idxA`、`idxB`，本质上就是 FPGA 找到的“峰值大概在哪个采样点”。

所以 `idxA`、`idxB` 是粗定位结果。

问题在于：

1. 如果噪声太大，最高点可能抖动。
2. 如果峰值不尖锐，最大点可能不好找。
3. 如果找错了一个点，后面整个 `dt` 都会偏。

这就是“峰值搜索误差”。

## 4. 什么叫“检测窗口”

“检测窗口”就是：

**你不是在整条数据里到处乱找峰值，而是在一个限定范围内找。**

这个限定范围就叫窗口。

你可以把它理解成：

“我知道目标大概会出现在某一段附近，所以只在这一小段里找最大值。”

这样做的好处：

1. 运算量更小
2. 不容易被远处无关噪声干扰

但它也会带来问题：

1. 窗口太小，真正峰值可能不在里面
2. 窗口太大，可能把别的假峰也算进来

所以“检测窗口选择不当”就是说：

**找峰值的范围没选好，导致真正的相关峰值没有被正确锁定。**

## 5. 什么叫“插值细化”

你在论文里看到的“插值细化”，本质上就是：

**峰值先粗略找到一个采样点，再进一步估计峰值其实落在这个点附近的哪个更精确的位置。**

因为采样是离散的。

例如：

- 真实峰值可能在 `10.3`
- 但采样点只有 `10`、`11`
- FPGA 只能先告诉你“峰值大概在第 10 个点附近”

这时候如果你只用整数采样点，误差会比较大。

所以就需要“插值细化”。

你的项目里用的是：

**三点抛物线插值**

也就是利用峰值点附近的三个点 `y1`、`y2`、`y3` 去拟合一个小抛物线，然后估计真正峰顶在什么位置。

这一步在你的系统里非常重要，因为它能把“整点级”的时间差估计，变成“亚采样级”的时间差估计。

## 6. 为什么插值也会带来误差

插值不是魔法，它只是“更聪明地猜”。

如果相关峰附近的三个点很理想，像一个平滑的小山峰，那插值效果很好。

但如果：

1. 三个点本身带噪声
2. 峰值不对称
3. 峰值很平，不够尖

那你拟合出来的抛物线峰顶位置就可能偏。

所以“插值细化误差”就是说：

**虽然做了插值，但插值得到的位置和真实峰值位置还是有差。**

## 7. 什么叫“固定点量化误差”

这个词听起来很硬件，其实本质很简单：

**数字系统没法表示无限精确的小数，所以总要取整、截断或舍入。**

例如：

- 真值是 `0.123456`
- 你的系统只保留 `0.123`

那中间被截掉的那一点，就是量化误差。

在 FPGA 或某些嵌入式实现里，这种误差会来自：

1. 数据位宽有限
2. 乘法后要截位
3. 除法结果要保留有限小数位

如果你后面真的去做“MATLAB 按硬件位宽模拟”，这个误差就会很值得写。

如果暂时不做，那你只需要理解成：

**数字计算不可能无限精确，有限位宽本身就会带来误差。**

## 8. 什么叫“模型参数误差”

这类误差不是 `dt` 没算出来，而是：

**你把 `dt` 换算成流速、流量时，用到的参数本身不准。**

例如：

1. 管道内径 `D` 不准
2. 安装角度 `θ` 不准
3. 声程长度 `L` 不准
4. 固定延时 `te` 不准
5. 管壁传播时间补偿不准

这时哪怕 `dt` 本身算得很好，最后的 `v` 和 `Q` 还是会偏。

所以这类误差更像是：

**后级换算模型本身带来的系统偏差。**

## 9. 为什么低流速时更容易出问题

这个非常关键，也很适合写进论文。

原因很简单：

低流速时，顺流和逆流传播时间差本来就很小。

也就是说：

1. 真正要测的 `dt` 本来就很小
2. 这时同样大小的噪声或采样误差，占比就会变大

举个很粗的例子：

- 如果真值 `dt = 100 ns`
- 误差是 `5 ns`
- 相对误差是 `5%`

但如果：

- 真值 `dt = 10 ns`
- 误差还是 `5 ns`
- 相对误差就变成 `50%`

所以低流速条件下，系统会显得更“脆弱”。

## 10. 什么叫“流动状态和安装条件误差”

这类误差比较工程化。

主要包括：

1. 实际流场不理想  
   比如弯头后面、阀门后面、局部紊流，都会让流速分布不均匀。

2. 换能器安装不理想  
   比如角度偏了、位置偏了、贴合不好。

3. 耦合状态变化  
   比如耦合剂不均匀、时间长了老化。

4. 管壁材质和厚度影响  
   不同材料里的传播速度不同，补偿不准也会带来偏差。

这些误差通常不太容易像噪声那样直接精确算出来，所以更适合先做定性分析，后面在实验或仿真里做部分敏感性分析。

## 11. 2.4 这一节到底在说什么

你可以把 `2.4 时间差测量中的主要误差来源` 理解成：

**“还没开始做实验之前，先从原理上把系统可能错在哪里想清楚。”**

它不是在这一节里就把所有误差都算完，而是在做误差分类和机理分析。

真正到了第六章，才更适合去做：

1. 噪声条件下误差统计
2. 插值前后误差对比
3. 参数扰动敏感性分析
4. `Bias`、`STD`、`RMSE` 等指标评估

## 12. 你现在最值得记住的简化版本

如果把 `2.4` 压缩成一句最容易记的话，就是：

**误差可能来自四个地方：信号本身不干净、峰值没找准、换算参数不准、安装和工况不理想。**

如果再压成更像项目链路的话，就是：

`信号进来会错 -> 找峰值会错 -> 插值会错 -> dt 会错 -> 参数换算也可能错 -> 最后流量就偏了`

这就是整节最核心的意思。

## 13. 和你当前代码怎么对应

如果和你现有代码对应，大概可以这样理解：

1. `idxA / idxB`
   是峰值粗定位，可能有搜索误差

2. `y1 / y2 / y3`
   是插值细化用的邻域点，可能受噪声影响

3. `t1 / t2 / dt`
   是后级解算结果，前面任一步错了都会传到这里

4. `te_ns`、`cos_value`、`sin_value`、`inner_diameter`
   是模型参数，参数不准会带来系统偏差

5. `flow_speed`、`flow_rate`
   是最终输出，前端误差和参数误差最终都会反映到这里

## 14. 你后面最适合做的验证

如果按这份解释继续往下做，第六章最适合对应的就是：

1. 噪声对 `dt` 的影响
2. 有无抛物线插值的对比
3. `D`、`θ`、`te` 的参数扰动分析
4. 低流速条件下误差放大分析

这四个做出来，`2.4` 和第六章就能前后呼应上。
