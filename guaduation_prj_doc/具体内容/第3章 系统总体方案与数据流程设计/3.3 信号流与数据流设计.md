# 3.3 信号流与数据流设计

为清晰说明系统内部处理逻辑，有必要区分信号流与数据流。前者关注超声信号从发射到接收的物理过程，后者关注采样结果进入数字处理链路后的组织与传递方式。二者相互衔接，但描述重点不同。

从信号流看，系统首先由 FPGA 发射控制模块按既定节拍产生 PWM 激励信号，经前端驱动电路作用于换能器后形成超声发射波。超声波沿预定声路传播并被接收换能器转换为电信号，再经接收前端放大与整形后输入 ADC。发射结束后，FPGA 发出同步采样触发信号，使 ADC 采样窗口与回波到达时刻保持对应关系。由此，系统中的信号流主线可概括为“激励产生—声波发射—介质传播—回波接收—模拟调理—数字采样”。

从数据流看，ADC 输出的采样结果首先进入 FPGA 内部缓存，随后在 FPGA 中完成卷积或相关处理、峰值搜索及特征提取。经过前级处理后，原始高数据量波形被压缩为能够支撑后级解算的关键特征，并以固定格式送入 MCU。MCU 仅对这些特征进行后续计算，而不再处理整段原始波形。由此，系统中的数据流主线可概括为“采样数据—FPGA 前级处理—特征数据包—MCU 后级解算—结果输出”。

从处理层次上看，数据流可分为原始采样数据、特征数据和测量结果三个阶段。前两者主要分布在 FPGA 内部，后一阶段则面向 MCU 侧结果生成与管理。这样的组织方式避免了平台间传输大量冗余波形数据，也使物理信号采集过程与数字处理过程能够在时序上紧密配合。

总体而言，信号流说明了超声测量链路中“信号如何产生并被采集”，数据流说明了“采样结果如何转化为可用于解算的特征”。将二者分开讨论，有助于明确系统结构层次，也为后续特征数据包组织及具体模块设计提供清晰入口。
