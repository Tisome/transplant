# 3.4 特征数据包设计

本系统中，FPGA 与 MCU 之间的数据交互围绕时间差解算所需的关键特征展开，而非围绕整段原始采样波形展开。FPGA 在内部完成卷积、峰值搜索和相关特征提取后，仅输出能够直接支撑后级解算的少量结果，并将其组织成固定格式的数据包发送给 MCU。这样既降低了跨平台传输压力，也使 MCU 的处理对象更加集中。

数据包由 `idxA`、`idxB`、`y1`、`y2`、`y3` 五部分组成。其中，`idxA` 与 `idxB` 分别表示两帧测量中相关峰的大致位置，`y1`、`y2`、`y3` 表示峰值邻域的三点相关结果，用于支撑后续抛物线插值。按照当前组织方式，`idxA` 和 `idxB` 各占 `2 Byte`，`y1`、`y2`、`y3` 各占 `6 Byte`，因此整包长度为 `22 Byte`。与直接上传整段波形相比，该结构显著压缩了数据量，同时保留了时间差细化所需的必要信息。

从时序上看，特征数据包并非在单帧处理完成后立即发送，而是在一组完整数据准备就绪后统一打包。系统先在两个 `4 ms` 子周期内获得两帧测量结果并提取索引信息，再由相关处理单元生成 `y1`、`y2` 和 `y3`。当上述特征全部齐备后，通信模块才将其组装为完整数据包并进入待发送状态。这样可以保证 MCU 每次读取到的都是一组完整的后级解算输入，从而简化接口侧状态管理。

在通信机制上，系统采用 MCU 为主机、FPGA 为从机的 SPI 传输方式。FPGA 完成数据整理后先缓存结果，再通过中断通知 MCU 进行读取。该方式使前端测量节拍与后端读取过程在逻辑上实现解耦，既保证了 FPGA 侧处理的连续性，也提高了接口组织的清晰性与稳定性。

这种数据包设计体现了系统前后级分工的基本思路：由 FPGA 负责从大数据量波形中提取关键特征，由 MCU 负责依据特征完成时间差细化和结果解算。该结构既避免了冗余数据传输，又为后级算法提供了稳定、统一的输入格式。
