# SPI8370模块设计文档

模块设计目标

本模块用于实现基于FPGA的SPI主机控制器，写入串行数据给AD模块进行增益数值控制，通过串行通信协议将8位增益控制字写入芯片，实现对超声波接收信号幅度的调整，使采样信号处于合适的量化范围。

## 二、架构设计

该模块以分频计数器为基础实现状态机控制逻辑，包括了时钟生成、数据缓冲、并行转串行三大功能

## 三、详细逻辑分析

### 3.1 时钟及计数控制

1. 分频计数器cnt：根据基准的系统时钟sys_clk进行预先设定的参数DIV进行分频，定义了半周期hlf和全周期clr信号实现spi所需的同步时钟sclk，其频率为系统时钟频率的1/DIV。在该模块中DIV参数预先设定为10，所以sclk的频率一般为5Mhz。

2. 比特计数器dnt：用三位计数器记录当前传输的比特位，其变化为从0-7，该计数器是在当req信号为高电平且ltch信号为低电平时随着sclk递增，在代码中用&dnt表示当dnt==3’b111(3’d7)时其在下个sclk时钟周期变为0，直到再次遇到req信号拉高才会进行变化

### 3.2 数据触发与缓冲

写使能捕捉：该模块监测wr_en信号。一旦出现高电平脉冲立刻将wr_dat锁存到模块内的dat寄存器上并拉高req信号。

在该模块实际的使用中，Top模块调用SPI模块并将wr_en定为1’b1，将wr_dat定为8’d99，这样做也是可以的，就是让dat保持为8’d99，其效果也就是连续不间断地输出8’d99。同时在这种情况下req和各控制信号可以正常运作。

### 3.3 串行接口时序

串行接口的时序问题主要分为ltch信号逻辑和mosi输出逻辑。

对于ltch，在写使能信号将req信号拉高之后，在sclk时钟的下降沿将ltch信号拉低，开启AD芯片的写入口，当八位数据发送完毕后将LTCH拉高，产生上升沿触发AD芯片逻辑使新的增益值生效。

Mosi输出采用组合逻辑，当ltch为1使，mosi持续输出0；当ltch为0时，其信号为dat[7-dnt]，前文提到过dnt为比特计数器，dat为内部数据寄存器，dnt在ltch为0且sclk时钟信号遇到下降沿时才自增，因此当ltch初次变为0的那个sclk时钟周期，dnt计数器保持不变，这样就能让mosi在ltch信号有效期间能够将八位数据从最高位到最低位映射到输出引脚。

## 四、主要信号

输入系统时钟频率：clk_50M：50Mhz

Sclk同步时钟频率：f_sclk = 50Mhz / DIV = 5Mhz （DIV默认为10）

传输位宽：8位

wr_en和wr_dat需要至少保持一个系统时钟周期使得可以被捕获

ltch型号在八位数据采集期间全程保持低电平
