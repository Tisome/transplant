# AD（数据采集）模块设计文档

模块设计目标

本模块实现对于外部高速ADC的数据采集和缓存功能，主要用于同步采集、存储和跨时钟域传输ADC数据并实现数据精度从12位到16位的扩展。

## 二、架构设计

该模块分为高速写入单元、低速读取单元和AD_RAM单元。其中，高速写入单元的功能为获取ad_start信号后采集原始AD数据、扩展该数据精度，生成数据写入地址并将数据存入AD_RAM单元；低速读取单元的功能为在写入完成之后生成地址并读取数据；AD_RAM单元负责存储数据并实现跨时钟域的读写。

## 三、详细逻辑分析

### 3.1 高速写入单元逻辑

1. 控制写入：该单元获取ad_start控制信号，在ad_start拉高之后打两拍持续拉高ad_en信号，ad_en控制AD_RAM端口A的写入。因此从该时刻起端口A可以写入数据。

2. 原始数据处理：当ad_start拉高时，外部的adc_data同步写入12位偏移二进制码，为了使得后续可以进行更高精度的处理，该单元中对其进行两步转换。首先将adc_data的最高位取反后写入adc_data_d0，该步将原始数据转换为二进制补码。其次扩展adc_data_d0的高位并重复四位后写入adc_data_wide。这两部相当于原始数据与ad_en同步打了两拍。

3. 写入地址生成：当ad_en信号拉高时，地址信号addra每个时钟周期自增1，直到addra==AD_LEN-1时地址变为0。同时ad_en拉低，直到下一次ad_start信号的到来。以上设计使得地址信号和经过处理后的数据信号可以同步写入AD_RAM。整体写入速率即为f_adc_clk = 65MHz

### 3.2 低速读取单元逻辑

低速读取单元的时钟为al_clk，其任务为生成读取地址和enb信号，获取AD_RAM端口B的数据并输出至外部其他模块。其生成地址及控制信号的逻辑为当addra==AD_LEN-1时，trans_flag在下个al_clk的时钟周期拉高并保持。从这时起，clk_cnt变量开始随着时钟周期自增1，当其达到49时重新从0开始自增1并循环。每当cnt_clk达到49时enb都拉高一个时钟单元，使得AD_RAM开始读取数据并使addrb自增1。

当addrb为AD_LEN-1且clk_cnt为49时，trans_flag拉低。addrb在变为2000后的下个时钟周期就回到0并保持。整个流程使得能够低速读取adc_buf_data。其读取速率为f_al_clk/(cnt+1) = 2Mhz。

### 3.3 AD_RAM存储单元逻辑

在Vivado中用IP catalog引入blk_mem_gen即可，端口A是写入端口，端口B是读取端口，数据宽度为16bit，数据容量应当超过2000，源代码中用的是6000+数据容量，可能是为了保险

### 3.4 逻辑问题

该模块获取的ad_start信号是从pwm模块中发出的，而在pwm模块中我们直到ad_start持续15ns。但是接受方写入单元的时钟用的是65MHz的adc_clk，其一个时钟周期约为15.384ns，所以其实是有一定概率整个ad_start信号为高电平期间没有遇到任何一个adc_clk的上升沿，这个据毛博说是打算要改成adc_clk为40MHz，具体的后续再说，先留个小疑问。
